Sistema de Lead Scoring
Score bidimensional (perfil + comportamento) com classificacao automatica e gatilhos de acao.
Implementado em: orchestrator._updateScore()
SCORE PERFIL (0-50 pts)
Baseado nos dados coletados do lead
Cada dado novo adiciona pontos ao perfil. O agente decide o incremento via score_update.perfil.
Exemplos: nome (+5), provedor (+10), cidade (+5), porte (+10), ERP (+5), num_clientes (+10), decisor (+5).
SCORE COMPORTAMENTO (0-50 pts)
Baseado nos sinais de engajamento
Cada interacao positiva incrementa o score. O agente decide via score_update.comportamento.
Exemplos: resposta rapida (+3), pediu demo (+15), perguntou preco (+10), mostrou dor (+8), engajamento alto (+5).
CLASSIFICACAO AUTOMATICA
score_total = perfil + comportamento (0-100)
Frio: 0-30
Lead recem-chegado ou desengajado. Pode ser devolvido para Sofia (nurturing).
Morno: 31-60
Lead engajado, respondendo perguntas, mostrando interesse. Carlos continua qualificando.
Quente: 61-80
Lead qualificado com dor clara. Gatilho para transferir_vendas → Lucas.
Ultra Quente: 81-100
Lead pronto para fechar. Gatilho para transferir_closer → Rafael.
_updateScore(leadId, scoreUpdate) { novoPerfil = Math.min(50, lead.score_perfil + scoreUpdate.perfil); novoComportamento = Math.min(50, lead.score_comportamento + scoreUpdate.comportamento); total = novoPerfil + novoComportamento; classificacao = total >= 81 ? 'ultra_quente' : total >= 61 ? 'quente' : total >= 31 ? 'morno' : 'frio'; }

Sistema de Skills & Knowledge Injection
Conhecimento estatico em markdown injetado no system prompt do agente, filtrado por agentKey + taskType.
Implementado em: skills-knowledge.js | Arquivos: skills-ref/*.md
ENTRADA
claude._getSystemPrompt(agentKey, context)
Ao compor o system prompt, chama skillsKnowledge.getCompactContext(agentKey, taskType)
1. INFER TASK TYPE
_inferTaskType(lastMessage)
Analisa a mensagem do lead e mapeia para 1 dos 12 tipos de tarefa usando keywords.
cold-email | email-sequence | ad-campaign | demand-gen | lead-research pricing | copywriting | strategy | sales | closing | orchestration | general
2. FILE MAPPING
Seleciona arquivos por agente
Cada agente tem 2-3 arquivos de skills mapeados:
AgenteArquivos Mapeados
Carlosvendas-sofia.md, prospecao-pricing.md, emailseq.md
Lucasvendas-sofia.md, prospecao-pricing.md
Rafaelvendas-sofia.md, prospecao-pricing.md
Sofiavendas-sofia.md, demandgen.md, prospecao-pricing.md
Leomarcos-leo.md, emailseq.md, demandgen.md
Marcosmarcos-leo.md, demandgen.md
Dianamarcos-leo.md, vendas-sofia.md, agentes-arch.md
3. EXTRACT SECTIONS
_extractSection(content, agentKey)
Extrai TODAS as secoes relevantes por header markers (regex multi-match).
Cada agente tem markers: ex. Carlos = ['CARLOS', 'PROSPECAO', 'PRINCIPIOS FUNDAMENTAIS']
Diana recebe o arquivo inteiro (sem filtro de secao).
4. FILTER BY TASK TYPE
getCompactContext() — filtra por keywords do taskType
Divide o conhecimento em secoes (por ##) e filtra as que contem keywords do tipo de tarefa.
Ex: taskType='pricing' filtra por ['PRICING', 'TIER', 'PRECO', 'VAN WESTENDORP', 'FREEMIUM', 'TRIAL']
Se taskType='general', retorna tudo (sem filtro).
5. INJETA NO PROMPT
Max 8000 chars, cache de 5 min
Resultado concatenado ao system prompt como \n\nBASE DE CONHECIMENTO (taskType):\n...
Sistema de Treinamento Continuo
Aprendizado automatico extraido de conversas de sucesso. Regras injetadas no system prompt.
Implementado em: training.js | Tabela: treinamento_agentes
GATILHO
Evento de avanco no funil
Dispara quando o orchestrator detecta uma das 4 acoes de sucesso:
['transferir_vendas', 'transferir_closer', 'agendar_demo', 'enviar_proposta']
ANALISE (Claude Sonnet)
training.analyzeConversation(agentKey, texto, outcome, client)
Fire-and-forget: roda async, nao bloqueia resposta ao lead.
Claude Sonnet analisa a conversa inteira e extrai aprendizados em JSON.
// Claude recebe: "Analise esta conversa do agente {agentKey} com resultado {outcome}" // Claude responde: { "aprendizados": [ { "tipo": "objecao_superada", "regra": "...", "contexto": "quando aplicar" }, { "tipo": "abordagem_eficaz", "regra": "...", "contexto": "..." } ] }
LEARN
training.learn(agentKey, tipo, regra, contexto)
Para cada aprendizado extraido:
- Se ja existe (agente+tipo+regra iguais): incrementa vezes_aplicada e confianca +0.05
- Se nao existe: INSERT com confianca=0.5, fonte='automatico'
INJECAO NO PROMPT
training.getTrainingContext(agentKey)
Busca top 15 regras ativas (confianca >= 0.3) agrupadas por tipo:
Objecoes superadas | Abordagens eficazes | Insights regionais | Insights de persona | Frases que convertem | Erros a evitar | Timing ideal
// Injetado como: \n\nAPRENDIZADOS DAS CONVERSAS ANTERIORES: Objecoes que funcionaram: - Lead de provedor pequeno responde melhor quando... Abordagens eficazes: - Mencionar o efeito de rede regional gera mais interesse...

Supervisor: Diana (Orquestracao Multi-Agente)
Diana nao executa — ela planeja, delega e consolida. Usada para demandas de marketing e coordenacao entre agentes.
Implementado em: supervisor.js | Routes: /api/supervisor/*
DEMANDA
POST /api/supervisor/run
Recebe demanda de alto nivel. Ex: "Criar campanha de lancamento para regiao Sul"
DIANA PLANEJA
createDemand(demanda, contexto)
Diana analisa a demanda e cria plano de execucao em JSON:
Cada tarefa tem: ordem, agente, tarefa (descricao), briefing, depende_de, prioridade.
// Diana retorna: { "plano_execucao": [ { "ordem": 1, "agente": "sofia", "tarefa": "Estrategia regional", "briefing": "...", "depende_de": null }, { "ordem": 2, "agente": "leo", "tarefa": "Criar copy ads", "briefing": "...", "depende_de": 1 }, { "ordem": 2, "agente": "marcos", "tarefa": "Configurar campanha", "briefing": "...", "depende_de": 1 }, { "ordem": 3, "agente": "diana", "tarefa": "Consolidar e lançar", "briefing": "...", "depende_de": 2 } ] }
EXECUTA PLANO
executePlan(taskId)
Agrupa tarefas por ordem. Executa mesma ordem em paralelo (Promise.all).
Tarefas com depende_de recebem resultado da tarefa anterior no briefing.
Ordens diferentes executam sequencialmente.

Executa sequencialmente
ORDEM 2 (PARALELO)
Leo + Marcos
Promise.all() — ambos em paralelo
DIANA CONSOLIDA
_consolidateResults(task)
Diana recebe todos os resultados, avalia qualidade de cada entrega, e gera relatorio consolidado com proximos passos.
RESULTADO
Retorna consolidacao + resultados detalhados + tokens totais

Esquema completo com 9 tabelas + indices. Arquivo: data/agentes.db

leads

Tabela principal do CRM
  • telefone (UNIQUE), nome, provedor, cidade, estado, regiao
  • porte, erp, num_clientes, decisor, email, cargo, site
  • score_perfil (0-50), score_comportamento (0-50), score_total
  • classificacao (frio/morno/quente/ultra_quente)
  • etapa_funil, agente_atual, origem, valor_estimado

conversas

Log de mensagens
  • lead_id (FK), agente, direcao (recebida/enviada)
  • mensagem, tipo, canal (whatsapp)
  • tokens_usados, tempo_resposta_ms
  • metadata (JSON: acao, notas)

atividades_agentes

Log de atividades
  • agente, tipo, descricao
  • lead_id, decisao, score_antes, score_depois
  • tokens_usados, tempo_ms

handoffs

Historico de transferencias
  • lead_id, de_agente, para_agente
  • motivo, score_no_momento

treinamento_agentes

Regras aprendidas
  • agente, tipo, regra, contexto, fonte
  • confianca (0.0-1.0), vezes_aplicada, vezes_sucesso
  • ativo (0/1)

metricas_diarias

KPIs por agente/dia
  • data + agente (UNIQUE)
  • mensagens_enviadas/recebidas, leads_novos/qualificados
  • demos_agendadas, propostas_enviadas, contratos_fechados
  • tokens_consumidos, valor_pipeline

tarefas

Fila de tarefas
  • lead_id, agente, tipo (demo/proposta/handoff)

campanhas

Campanhas de prospeccao
  • nome, tipo, agente, regiao, status
  • total_enviados, total_respondidos, total_qualificados

sessoes_agentes

Sessoes ativas
  • lead_id, agente, session_id, ativo
