### 1.3 — Busca ampliada por Nome e Telefone

O campo de busca já aceita CPF/CNPJ/CEP. Adicione suporte a:

- **Nome parcial**: Se o texto tiver 5+ letras e começar com letra, detectar como "Nome" e fazer busca LIKE no campo `nome` da tabela `inadimplentes`
- **Telefone**: Se tiver 10-11 dígitos numéricos, detectar como "Telefone" e buscar no campo `telefone`

No backend, adicionar lógica de detecção no endpoint `POST /api/isp-consultations`:
```typescript
// Detectar tipo do documento recebido
const docLimpo = documento.replace(/\D/g, '')
if (docLimpo.length === 11) tipo = 'CPF'
else if (docLimpo.length === 14) tipo = 'CNPJ'  
else if (docLimpo.length === 8) tipo = 'CEP'
else if (docLimpo.length >= 10 && docLimpo.length <= 11) tipo = 'TELEFONE'
else if (documento.length >= 5 && /^[a-zA-ZÀ-ÿ]/.test(documento)) tipo = 'NOME'
```

Para busca por nome/telefone, retornar resultados sem cobrar créditos (busca gratuita na base interna).

  cpf_cnpj,

## PARTE 3 — NOVA FUNCIONALIDADE: IMPORTAR EQUIPAMENTOS PERDIDOS

### Por que esta funcionalidade é necessária

A maioria dos provedores não registra equipamentos perdidos no ERP. Isso significa que quando outro provedor consulta um CPF no Consulta ISP, o sistema não sabe que aquele cliente tem ONU ou roteador retido — e não alerta corretamente. Esta ferramenta permite registrar equipamentos via planilha, sem precisar de ERP.

### 3.1 — Nova entrada no menu lateral

Adicionar no sidebar, na seção "GESTÃO", um novo link:
- Ícone: 📦 (ou `Package` do lucide-react)
- Label: "Importar Equip."
- Rota: `/importacao-equipamentos`

### 3.2 — Backend: tabela e endpoints de equipamentos

**Verificar se a tabela `equipamentos` já existe** no banco. Se não existir, criar:
```sql
CREATE TABLE IF NOT EXISTS equipamentos (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  provider_id INTEGER NOT NULL,
  inadimplente_id INTEGER,
  cpf_cnpj TEXT NOT NULL,
  nome_cliente TEXT,
  tipo TEXT DEFAULT 'ONU',
  marca TEXT,
  modelo TEXT,
  numero_serie TEXT,
  valor REAL DEFAULT 0,
  data_perda TEXT,
  observacao TEXT,
  status TEXT DEFAULT 'retido',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (provider_id) REFERENCES providers(id)
);
```

**Criar arquivo `server/routes/equipamentos.ts`** (ou `.js`) com:

```typescript
// GET /api/equipamentos — listar equipamentos do provedor
router.get('/', auth, (req, res) => {
  const db = getDb()
  const items = db.prepare(`
    SELECT e.*, i.nome as nome_inadimplente 
    FROM equipamentos e 
    LEFT JOIN inadimplentes i ON e.cpf_cnpj = i.cpf_cnpj AND i.provider_id = e.provider_id
    WHERE e.provider_id = ? 
    ORDER BY e.created_at DESC
  `).all(req.user.provider_id)
  
  const stats = {
    total: items.length,
    retidos: items.filter(e => e.status === 'retido').length,
    recuperados: items.filter(e => e.status === 'recuperado').length,
    valor_risco: items.filter(e => e.status === 'retido').reduce((s, e) => s + (e.valor || 0), 0)
  }
  
  res.json({ items, stats })
})

// POST /api/equipamentos — cadastrar um equipamento
router.post('/', auth, (req, res) => {
  const { cpf_cnpj, nome_cliente, tipo, marca, modelo, numero_serie, valor, data_perda, observacao } = req.body
  if (!cpf_cnpj) return res.status(400).json({ message: 'CPF/CNPJ obrigatório' })
  
  const db = getDb()
  const r = db.prepare(`
    INSERT INTO equipamentos 
    (provider_id, cpf_cnpj, nome_cliente, tipo, marca, modelo, numero_serie, valor, data_perda, observacao)
    VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
  `).run(req.user.provider_id, cpf_cnpj, nome_cliente, tipo || 'ONU', marca, modelo, numero_serie, valor || 0, data_perda, observacao)
  
  res.json({ id: r.lastInsertRowid, message: 'Equipamento cadastrado' })
})

// POST /api/equipamentos/import — importar múltiplos via array (CSV parsed)
router.post('/import', auth, (req, res) => {
  const { rows } = req.body
  if (!rows?.length) return res.status(400).json({ message: 'Dados obrigatórios' })
  
  const db = getDb()
  let ok = 0
  const erros: Array<{ linha: number; erro: string }> = []
  
  const ins = db.prepare(`
    INSERT INTO equipamentos (provider_id, cpf_cnpj, nome_cliente, tipo, marca, modelo, numero_serie, valor, data_perda, observacao)
    VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
  `)
  
  rows.forEach((row: any, i: number) => {
    if (!row.cpf_cnpj) {
      erros.push({ linha: i + 2, erro: 'CPF/CNPJ ausente' })
      return
    }
    try {
      ins.run(
        req.user.provider_id, 
        row.cpf_cnpj, 
        row.nome_cliente || null, 
        row.tipo_equipamento || 'ONU',
        row.marca || null, 
        row.modelo || null, 
        row.numero_serie || null,
        parseFloat(row.valor_equipamento) || 0,
        row.data_perda || null,
        row.observacao || null
      )
      ok++
    } catch (e: any) {
      erros.push({ linha: i + 2, erro: e.message })
    }
  })
  
  res.json({ importados: ok, erros })
})

// PATCH /api/equipamentos/:id — atualizar status
router.patch('/:id', auth, (req, res) => {
  const db = getDb()
  db.prepare('UPDATE equipamentos SET status = ? WHERE id = ? AND provider_id = ?')
    .run(req.body.status || 'retido', req.params.id, req.user.provider_id)
  res.json({ message: 'Atualizado' })
})

// DELETE /api/equipamentos/:id
router.delete('/:id', auth, (req, res) => {
  const db = getDb()
  db.prepare('DELETE FROM equipamentos WHERE id = ? AND provider_id = ?')
    .run(req.params.id, req.user.provider_id)
  res.json({ message: 'Removido' })
})
```

Registrar a rota no `server.ts`/`index.ts` principal:
```typescript

app.use('/api/equipamentos', equipamentosRoutes)
```
