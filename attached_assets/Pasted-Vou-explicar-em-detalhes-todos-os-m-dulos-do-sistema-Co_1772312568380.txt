Vou explicar em detalhes todos os módulos do sistema Consulta ISP:

🏠 MÓDULOS PRINCIPAIS
1. Dashboard
Painel central com visão geral do sistema
KPIs principais: total de clientes, inadimplentes, valores em atraso
Mapa de calor mostrando concentração de inadimplentes por região
Alertas de fraude recentes
Atalhos rápidos para ações comuns
2. Consulta ISP
Consulta clientes em uma base de dados colaborativa entre provedores
Verifica histórico de pagamento de clientes em outros provedores
Mostra score de risco do cliente
Indica se o cliente tem pendências, equipamentos não devolvidos
Ajuda na decisão de aprovar ou rejeitar novos clientes
3. Consulta SPC
Integração com bureaus de crédito (SPC/Serasa)
Consulta situação cadastral do CPF/CNPJ
Mostra restrições financeiras, protestos, cheques sem fundo
Gera relatórios em PDF
Consome créditos do provedor
4. Anti-Fraude
Detecta quando seus clientes são consultados por outros provedores
Alerta sobre possíveis "golpes" (cliente cancela e vai para concorrente devendo)
Score de risco calculado por IA
Análise de padrões de fraude recorrentes
Configuração de regras personalizadas
Notificações automáticas para alertas críticos
👥 GESTÃO DE INADIMPLENTES
5. Inadimplentes
Lista todos os clientes com faturas em atraso
Filtros por dias de atraso, valor, status
Detalhes completos de cada cliente
Histórico de interações e cobranças
Ações de cobrança (email, SMS, WhatsApp)
6. Mapa de Calor
Visualização geográfica dos inadimplentes
Identifica bairros/regiões problemáticas
Configuração de alertas por região
Ajuda no planejamento de visitas técnicas
💰 FINANCEIRO
7. Comprar Créditos
Recarregar saldo para consultas ISP e SPC
Histórico de transações
Pacotes de créditos disponíveis
Integração com gateway de pagamento
🛠️ FERRAMENTAS
8. Importação
Importar dados do seu ERP (IXC, Voalle, SGP, RBx, HubSoft, MK Solutions)
Upload de planilhas CSV/Excel
Sincronização automática de clientes, contratos, faturas e equipamentos
Histórico de importações
9. Administração
Gerenciar usuários do sistema
Definir papéis e permissões
Configurar integração com ERP
Configurações gerais do provedor
Gestão de equipamentos (tipos, modelos, fabricantes)
🛡️ ADMINISTRAÇÃO SAAS (Apenas para superadmin)
10. Painel SaaS
Visão geral de todos os provedores cadastrados
Métricas de uso da plataforma
Gerenciamento de tenants
11. CRM de Vendas
Pipeline de leads (provedores interessados)
Gestão de oportunidades comerciais
Automação de follow-up
12. API & Integrações
Configurar ERPs integrados
Monitorar jobs de importação
Logs de API
13. Gateway de Pagamento
Configurar métodos de pagamento
Visualizar transações
Webhooks de pagamento
14. Gestão Financeira
Receitas e faturamento
Previsões financeiras
Relatórios de cobrança
15. Deploy Manager
Gerenciar subdomínios dos provedores
Configurar DNS
Status de deploys
16. NFSe
Emissão de notas fiscais de serviço
Consulta e cancelamento de NFSe
17. Logo & Marca
Assets visuais do sistema
Logos para download
📊 ENTIDADES DE DADOS
Entidade	Descrição
Customer	Clientes dos provedores
Contract	Contratos de serviço
Invoice	Faturas/boletos
Equipment	Equipamentos em comodato
Provider	Provedores cadastrados
ISP_Consultation	Consultas ISP realizadas
SPC_Consultation	Consultas SPC realizadas
AntiFraudAlert	Alertas de fraude
User	Usuários do sistema
🔑 RESUMO DO FLUXO
Provedor se cadastra → Configura integração com ERP
Importa dados → Clientes, contratos, faturas, equipamentos
Monitora inadimplentes → Dashboard e lista de clientes
Recebe alertas → Quando seus clientes são consultados por outros
Consulta novos clientes → ISP (histórico) e SPC (crédito)
Toma ações → Cobrança, recuperação de equipamentos
O sistema é multi-tenant, ou seja, cada provedor vê apenas seus próprios dados, com isolamento total entre eles.
