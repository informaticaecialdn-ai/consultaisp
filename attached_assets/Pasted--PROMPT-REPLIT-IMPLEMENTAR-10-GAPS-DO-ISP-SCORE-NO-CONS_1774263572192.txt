## GAP 1 — CRÍTICO: Fluxo de Notificação Pré-Negativação (LGPD + CDC)

**Por que:** O CDC art. 43 §2 exige que o devedor seja notificado com mínimo 10 dias corridos antes de ser incluído em qualquer banco de inadimplência. Sem esse fluxo, registros no sistema podem ser contestados judicialmente.

### Backend

No endpoint `POST /api/inadimplentes` (arquivo `server/routes/inadimplentes.ts` ou equivalente), adicionar validação:

```typescript
// Novos campos obrigatórios no body:
// notificacao_enviada: boolean
// notificacao_data: string (ISO date)
// notificacao_canal: 'carta' | 'email' | 'whatsapp'

// Validação: data de notificação deve ser >= 10 dias antes de hoje
if (body.notificacao_enviada) {
  const diasDesdeNotif = Math.floor(
    (Date.now() - new Date(body.notificacao_data).getTime()) / 86400000
  );
  if (diasDesdeNotif < 10) {
    return res.status(400).json({
      error: 'NOTIFICACAO_PRAZO',
      message: `Aguarde ${10 - diasDesdeNotif} dia(s). O prazo mínimo de 10 dias corridos após a notificação ainda não foi atingido (CDC art. 43 §2).`
    });
  }
}
```

Adicionar colunas na tabela `inadimplentes` (migration):
```sql
ALTER TABLE inadimplentes ADD COLUMN notificacao_enviada INTEGER DEFAULT 0;
ALTER TABLE inadimplentes ADD COLUMN notificacao_data TEXT;
ALTER TABLE inadimplentes ADD COLUMN notificacao_canal TEXT DEFAULT 'whatsapp';
```

## GAP 3 — Histórico de Consultas do CPF por Outros ISPs

**O que é:** Quando um CPF é consultado, mostrar "Este CPF foi consultado X vezes por Y provedores diferentes nos últimos 30 dias."

### Backend

Criar tabela de log de consultas (se não existir):
```sql
CREATE TABLE IF NOT EXISTS consultation_logs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  cpf_cnpj TEXT NOT NULL,
  provider_id INTEGER NOT NULL,
  tipo TEXT DEFAULT 'isp',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX IF NOT EXISTS idx_conslog_cpf ON consultation_logs(cpf_cnpj, created_at);
```

No endpoint `POST /api/isp-consultations`, após a consulta, registrar no log e retornar estatísticas:
```typescript
// Registrar consulta
db.prepare('INSERT INTO consultation_logs (cpf_cnpj, provider_id) VALUES (?,?)').run(cpfCnpj, providerId);

// Buscar histórico de consultas por OUTROS provedores nos últimos 30 dias
const trinta_dias_atras = new Date(Date.now() - 30 * 86400000).toISOString();
const historicoConsultas = db.prepare(`
  SELECT
    COUNT(*) as total_consultas,
    COUNT(DISTINCT provider_id) as total_provedores,
    MIN(created_at) as primeira_consulta,
    MAX(created_at) as ultima_consulta
  FROM consultation_logs
  WHERE cpf_cnpj = ? AND provider_id != ? AND created_at >= ?
`).get(cpfCnpj, providerId, trinta_dias_atras);

// Adicionar ao resultado da consulta:
resultado.historico_consultas = {
  ultimos_30_dias: historicoConsultas.total_consultas,
  provedores_distintos: historicoConsultas.total_provedores,
  primeira_consulta: historicoConsultas.primeira_consulta,
  ultima_consulta: historicoConsultas.ultima_consulta,
  alerta: historicoConsultas.total_consultas >= 3
};
```

## GAP 4 — Histórico Financeiro do Cliente (Linha do Tempo)

**O que é:** Na tela de detalhes de um inadimplente, mostrar a linha do tempo do comportamento no setor: quantas vezes foi registrado, quando quito, padrão de atraso.

### Backend

Criar endpoint `GET /api/inadimplentes/:cpfCnpj/historico-rede`:
```typescript
router.get('/:cpfCnpj/historico-rede', auth, (req, res) => {
  const db = getDb();
  const { cpfCnpj } = req.params;
  const cpfLimpo = cpfCnpj.replace(/\D/g, '');

  // Histórico da própria base do provedor
  const historicoLocal = db.prepare(`
    SELECT
      id, name, total_overdue_amount, max_days_overdue,
      risk_tier, erp_source, last_sync_at, created_at,
      'proprio' as origem
    FROM inadimplentes
    WHERE provider_id = ? AND REPLACE(REPLACE(cpf_cnpj, '.', ''), '-', '') = ?
    ORDER BY created_at ASC
  `).all(req.user.provider_id, cpfLimpo);

  // Registros anonimizados de outros provedores na rede
  const historicoRede = db.prepare(`
    SELECT
      COUNT(*) as total_registros,
      MIN(max_days_overdue) as min_dias,
      MAX(max_days_overdue) as max_dias,
      AVG(max_days_overdue) as media_dias,
      MIN(created_at) as primeiro_registro,
      MAX(created_at) as ultimo_registro,
      SUM(CASE WHEN risk_tier = 'critical' THEN 1 ELSE 0 END) as casos_criticos
    FROM inadimplentes
    WHERE provider_id != ? AND REPLACE(REPLACE(cpf_cnpj, '.', ''), '-', '') = ?
  `).get(req.user.provider_id, cpfLimpo);

  res.json({
    historico_local: historicoLocal,
    historico_rede: historicoRede,
    perfil_risco: historicoRede.total_registros > 2 ? 'reincidente_grave'
      : historicoRede.total_registros > 0 ? 'reincidente'
      : 'sem_historico_rede',
    padrao_atraso: historicoRede.media_dias > 90 ? 'cronico'
      : historicoRede.media_dias > 60 ? 'grave'
      : historicoRede.media_dias > 30 ? 'moderado' : 'leve'
  });
});
```

## GAP 5 — Restrição Automática via Webhook do ERP

**O que é:** Quando o ERP bloqueia um cliente por inadimplência, registrar automaticamente no Consulta ISP sem ação manual.

### Backend

Criar endpoint público de webhook `POST /api/webhooks/erp-inadimplente`:

```typescript
// Arquivo: server/routes/webhooks.ts
import crypto from 'crypto';

router.post('/erp-inadimplente', (req, res) => {
  const db = getDb();

  // Autenticação via webhook_secret do provedor
  const authHeader = req.headers['x-webhook-secret'] as string;
  if (!authHeader) return res.status(401).json({ error: 'Missing webhook secret' });

  // Encontrar provedor pelo webhook_secret
  const provider = db.prepare(
    'SELECT * FROM providers WHERE webhook_secret = ? AND webhook_ativo = 1'
  ).get(authHeader);

  if (!provider) return res.status(401).json({ error: 'Invalid webhook secret' });

  const { cpf_cnpj, nome, valor_aberto, dias_atraso, telefone, cidade, estado } = req.body;

  if (!cpf_cnpj || !valor_aberto) {
    return res.status(400).json({ error: 'cpf_cnpj e valor_aberto são obrigatórios' });
  }

  // Verificar se já existe
  const existing = db.prepare(
    'SELECT id FROM inadimplentes WHERE provider_id = ? AND cpf_cnpj = ?'
  ).get(provider.id, cpf_cnpj);

  if (existing) {
    // Atualizar registro existente
    db.prepare(`
      UPDATE inadimplentes SET
        total_overdue_amount = ?, max_days_overdue = ?,
        last_sync_at = CURRENT_TIMESTAMP, erp_source = 'webhook'
      WHERE id = ?
    `).run(valor_aberto, dias_atraso || 0, existing.id);
    return res.json({ status: 'updated', id: existing.id });
  }

  // Inserir novo
  const result = db.prepare(`
    INSERT INTO inadimplentes
    (provider_id, name, cpf_cnpj, total_overdue_amount, max_days_overdue,
     phone, city, state, erp_source, notificacao_enviada, last_sync_at)
    VALUES (?,?,?,?,?,?,?,?,'webhook',0,CURRENT_TIMESTAMP)
  `).run(provider.id, nome || 'Não informado', cpf_cnpj, valor_aberto, dias_atraso || 0, telefone, cidade, estado);

  res.json({ status: 'created', id: result.lastInsertRowid });
});
```

Adicionar colunas na tabela `providers` para configuração do webhook:
```sql
ALTER TABLE providers ADD COLUMN webhook_secret TEXT;
ALTER TABLE providers ADD COLUMN webhook_ativo INTEGER DEFAULT 0;
```

Gerar webhook_secret automático para cada provedor:
```typescript
// Na criação de um provedor ou em migration:
const secret = crypto.randomBytes(32).toString('hex');
db.prepare('UPDATE providers SET webhook_secret = ? WHERE webhook_secret IS NULL').run(secret);
```

## GAP 8 — Trial de 30 dias do plano mais completo

### Backend

Na tabela `providers`, adicionar:
```sql
ALTER TABLE providers ADD COLUMN trial_inicio DATETIME DEFAULT CURRENT_TIMESTAMP;
ALTER TABLE providers ADD COLUMN trial_ativo INTEGER DEFAULT 1;
ALTER TABLE providers ADD COLUMN plano TEXT DEFAULT 'trial';
```

Criar endpoint `GET /api/provider/trial-status`:
```typescript
router.get('/trial-status', auth, (req, res) => {
  const db = getDb();
  const provider = db.prepare('SELECT trial_inicio, trial_ativo, plano FROM providers WHERE id = ?')
    .get(req.user.provider_id);

  const diasDecorridos = Math.floor(
    (Date.now() - new Date(provider.trial_inicio).getTime()) / 86400000
  );
  const diasRestantes = Math.max(0, 30 - diasDecorridos);

  res.json({
    trial_ativo: diasRestantes > 0,
    dias_restantes: diasRestantes,
    dias_decorridos: diasDecorridos,
    plano_atual: provider.plano,
    trial_expirou: diasRestantes === 0 && provider.plano === 'trial'
  });
});
```
