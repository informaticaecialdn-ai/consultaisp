  <action>
    Create server/erp/connectors/hubsoft.ts implementing ErpConnector. Per D-08:

    **OAuth2 Token Management:**
    - Private `tokenCache: Map<string, { token: string, expiresAt: number }>` keyed by apiUrl
    - Private `getAccessToken(config: ErpConnectionConfig): Promise<string>`:
      - Check cache: if token exists and expiresAt > Date.now() + 60000 (1 min buffer), return cached
      - POST to `${config.apiUrl}/oauth/token` with body: `{ grant_type: "password", client_id: config.extra.clientId, client_secret: config.extra.clientSecret, username: config.apiUser, password: config.apiToken }`
      - Content-Type: application/json (or application/x-www-form-urlencoded if Hubsoft requires — try JSON first)
      - Parse response: `{ access_token, token_type, expires_in }`
      - Cache with expiresAt = Date.now() + (expires_in * 1000)
      - Return access_token
    - Private `clearToken(apiUrl: string)`: removes cached token (for retry on 401)

    **testConnection:** Call getAccessToken — if it succeeds, connection works. 8s timeout on the OAuth call.

    **fetchDelinquents:**
    - Get Bearer token via getAccessToken
    - GET/POST `${config.apiUrl}/api/financeiro/inadimplentes` (or equivalent Hubsoft endpoint) with `Authorization: Bearer ${token}`
    - If 401 response, clearToken and retry once (token may have expired between cache check and use)
    - Map response to NormalizedErpCustomer using normalize helpers. Set erpSource: "hubsoft".

    **fetchCustomers:** Similar to fetchDelinquents but using clientes endpoint.

    **configFields:** `[{ key: "apiUser", label: "Username", type: "text", required: true }, { key: "apiToken", label: "Password", type: "password", required: true }, { key: "extra.clientId", label: "Client ID", type: "text", required: true }, { key: "extra.clientSecret", label: "Client Secret", type: "password", required: true }]`

    **Resilience:** Own CircuitBreaker instance. Wrap data fetch calls (not token calls) in withResilience. Token fetch failures should propagate immediately (not retried multiple times — the credentials are either right or wrong).

    NOTE: Hubsoft's exact endpoint paths may vary. Use best-guess from docs (https://docs.hubsoft.com.br/). The connector should handle both JSON and form-urlencoded token requests gracefully. If the first format fails with a parse error, try the other.
  </action>
