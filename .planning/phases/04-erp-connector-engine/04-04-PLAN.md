    **server/routes/erp.routes.ts** — Replace inline ERP logic with connector registry:

    1. Replace `import { testErpConnection, fetchErpCustomers } from "./utils"` with `import { getConnector, getSupportedSources } from "../erp"`
    2. In PATCH /:source endpoint: replace hardcoded `validSources` array with `const validSources = [...getSupportedSources(), "manual"]` — dynamically derived from registry
    3. In POST /:source/test endpoint:
       - Get connector: `const connector = getConnector(source)`
       - If !connector, return 400 "ERP nao suportado"
       - Build ErpConnectionConfig from integration record: `{ apiUrl: intg.apiUrl, apiToken: intg.apiToken, apiUser: intg.apiUser || "", extra: intg.extraConfig || {} }`
       - For Hubsoft, also populate extra.clientId and extra.clientSecret from the new schema fields
       - Call `connector.testConnection(config)` and return result
    4. In POST /:source/sync endpoint:
       - Same connector lookup and config building
       - Call `connector.fetchDelinquents(config)` instead of `fetchErpCustomers(...)`
       - The fetchResult shape matches: `{ ok, message, customers }` — the rest of the sync logic (storage.syncErpCustomers, logging, status updates) stays the same
    5. Keep the webhook endpoint (POST /api/webhooks/erp-sync) exactly as-is — per D-15, backward compat until Phase 5

    **server/routes/utils.ts** — Clean up:
    - Remove `testErpConnection` and `fetchErpCustomers` functions (they are now replaced by connectors)
    - Keep the other utility functions (getOverdueAmountRange, getRecommendedActions, score-engine re-exports)

    **server/scheduler.ts** — Replace hardcoded fetchErpCustomersForScheduler:
    1. Remove the entire `fetchErpCustomersForScheduler` function
    2. Import `{ getConnector }` from `"./erp"`
    3. Import `{ getProviderLimiter }` from `"./erp"` (rate limiting per provider)
    4. In `runAutoSync`, for each integration:
       - `const connector = getConnector(intg.erpSource)`
       - If !connector, log warning and skip (unknown ERP)
       - Build ErpConnectionConfig from integration fields (same pattern as routes)
       - `const limiter = getProviderLimiter(intg.providerId, intg.erpSource)`
       - `const fetchResult = await limiter(() => connector.fetchDelinquents(config))`
       - The rest of the sync logic (storage calls, logging) stays the same
    5. Replace console.log/console.warn with the same pattern (or keep as-is — pino migration is Phase 2 scope)

    IMPORTANT: The config building logic needs to handle the new Hubsoft OAuth fields. Create a small helper function `buildConnectorConfig(intg: ErpIntegration): ErpConnectionConfig` that:
    - Sets apiUrl, apiToken, apiUser from the integration record
    - If intg.clientId or intg.clientSecret exist, puts them in extra
    - If intg.extraConfig exists (jsonb), merges it into extra
    - This helper can live in server/erp/index.ts or in a shared location

    After changes, verify the app still builds: `npm run build`
