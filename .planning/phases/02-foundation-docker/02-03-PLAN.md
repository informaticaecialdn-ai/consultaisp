    **Dockerfile** (per D-06, D-07):
    Use node:20-slim (NOT alpine — per D-06, pg native bindings need glibc).

    ```dockerfile
    # Stage 1: Build
    FROM node:20-slim AS builder
    WORKDIR /app

    # Install build dependencies (tsx needs this)
    COPY package.json package-lock.json ./
    RUN npm ci

    # Copy source and build
    COPY . .
    RUN npm run build

    # Stage 2: Production
    FROM node:20-slim AS production
    WORKDIR /app

    # Copy built artifacts
    COPY --from=builder /app/dist ./dist
    COPY --from=builder /app/package.json ./package.json
    COPY --from=builder /app/package-lock.json ./package-lock.json

    # Install production dependencies only
    RUN npm ci --omit=dev

    # Set environment
    ENV NODE_ENV=production
    ENV PORT=5000
    EXPOSE 5000

    # Health check using the /api/health endpoint (created by Plan 02)
    HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
      CMD node -e "fetch('http://localhost:5000/api/health').then(r => r.ok ? process.exit(0) : process.exit(1)).catch(() => process.exit(1))"

    # Run as non-root
    USER node

    CMD ["node", "dist/index.cjs"]
    ```

    IMPORTANT: The build stage uses `npm ci` (full deps including dev) because `tsx` and `vite` are dev dependencies needed for the build. The production stage uses `npm ci --omit=dev` to exclude dev deps.

    IMPORTANT: Check if the build output includes frontend static files in dist/. The vite build in script/build.ts runs `viteBuild()` which outputs to the default vite output dir. Verify where frontend assets end up and ensure they're in the COPY from builder. If vite outputs to `dist/public/` or similar, the `COPY --from=builder /app/dist ./dist` already covers it. If vite outputs elsewhere (e.g., `client/dist/`), add an additional COPY line.

    **docker-compose.yml** (per D-08):
    ```yaml
    services:
      app:
        build: .
        ports:
          - "${PORT:-5000}:5000"
        environment:
          - DATABASE_URL=postgresql://consulta:${DB_PASSWORD}@postgres:5432/consultaisp
          - SESSION_SECRET=${SESSION_SECRET}
          - NODE_ENV=production
          - LOG_LEVEL=${LOG_LEVEL:-info}
          - PORT=5000
        env_file:
          - .env
        depends_on:
          postgres:
            condition: service_healthy
        restart: unless-stopped

      postgres:
        image: postgres:16-alpine
        volumes:
          - pgdata:/var/lib/postgresql/data
        environment:
          - POSTGRES_DB=consultaisp
          - POSTGRES_USER=consulta
          - POSTGRES_PASSWORD=${DB_PASSWORD}
        healthcheck:
          test: ["CMD-SHELL", "pg_isready -U consulta -d consultaisp"]
          interval: 5s
          timeout: 3s
          retries: 5
        ports:
          - "127.0.0.1:5432:5432"
        restart: unless-stopped

    volumes:
      pgdata:
    ```

    Key decisions:
    - PostgreSQL uses postgres:16-alpine (Alpine is fine for PG, only Node needs slim per D-06).
    - PG port exposed only on localhost (127.0.0.1) for security — no external access.
    - App depends_on postgres with condition: service_healthy so it waits for PG to be ready.
    - Persistent volume `pgdata` for database data.
    - env_file loads .env for secrets (RESEND_API_KEY, ASAAS_API_KEY, etc.).
    - DATABASE_URL is constructed inline pointing to the postgres service.

    **.env.example**:
